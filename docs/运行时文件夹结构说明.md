# 运行时文件夹存储规范（极简版）

来源：摘自《[需求分析与初步架构设计.md](/Users/jsuikak/Desktop/course_person_retrieve/docs/需求分析与初步架构设计.md)》中的“系统运行目录结构（极简版）”。

当前阶段不考虑测试与 manifest，仅用于直观组织“查询图、图库/视频库、索引、运行结果”。

```text
project_root/
├── data_runtime/
│   ├── query/                 # 查询图（search-image/search-video 的 --query）
│   └── gallery/
│       ├── images/            # 图像库原始数据
│       └── videos/            # 视频库原始数据
├── indexes/
│   ├── image_index/           # 图像索引目录（--index-dir）
│   └── video_index/           # 视频索引目录（--index-dir）
├── outputs/
│   ├── retrieval/             # 检索结果 JSON
│   └── annotate/              # 标注视频输出
├── src/
└── docs/
```

## 路径职责与参数映射

1. 查询图：`data_runtime/query/`
   - 对应参数：`search-image/search-video` 的 `--query`
2. 图像库：`data_runtime/gallery/images/`
3. 视频库：`data_runtime/gallery/videos/`
4. 图像索引：`indexes/image_index/`
   - 对应参数：`build-image-index/search-image/annotate-video` 的 `--index-dir`
5. 视频索引：`indexes/video_index/`
   - 对应参数：`build-video-index/search-video` 的 `--index-dir`
6. 检索结果：`outputs/retrieval/`
7. 标注视频：`outputs/annotate/`
   - 对应参数：`annotate-video` 的 `--output-video`

## 最小落地示例

1. 查询图路径：`data_runtime/query/ikura_0001.jpg`
2. 图像库目录：`data_runtime/gallery/images/`
3. 视频库目录：`data_runtime/gallery/videos/`
4. 图像索引目录：`indexes/image_index/`
5. 视频索引目录：`indexes/video_index/`
6. 检索结果目录：`outputs/retrieval/`
7. 标注输出路径：`outputs/annotate/short_annotated.mp4`
