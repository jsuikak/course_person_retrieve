## 3. 建议你做的实验矩阵（按优先级）

## 3.1 实验 A：基础性能对比（必须做）

目标：证明不同分支各自能力与短板。

设置：
1. A1: `face-only`（`feature_mode=face`）
2. A2: `person-only`（`feature_mode=person`）
3. A3: `face + person` 融合（如果来得及做一个线性融合/重排）

指标：
1. `Rank-1`
2. `Recall@5`, `Recall@10`
3. `mAP`

建议数据协议：
1. `yoasobi_TFT`：每身份至少切出 10 张 query（从原身份文件夹重划分，不只用现有 3 张）
2. `egodeath`：每身份按 `gallery:query = 7:3` 或 `8:2` 分层抽样
3. 固定随机种子，重复 3 次划分，报告均值和标准差

---

## 3.2 实验 B：鲁棒性实验（非常加分）

目标：说明你的系统在真实视频劣化场景下是否稳定。

对 query 人工退化：
1. 高斯模糊（例如 kernel=3,5,7）
2. 分辨率下降（1.0x, 0.5x, 0.25x）
3. 遮挡（眼部/口鼻区域遮挡）
4. 亮度扰动（暗光、过曝）

对比项：
1. `face-only` 精度下降曲线
2. `person-only` 精度下降曲线
3. 融合后下降幅度是否更小

答辩价值：可直接回答“你系统为什么比单一人脸更鲁棒”。

---

## 3.3 实验 C：检索参数消融（容易做，性价比高）

目标：证明你的参数不是拍脑袋。

建议消融变量：
1. `topk`: 1/5/10
2. 视频采样率 `sample_fps`: 0.5/1/2
3. 匹配阈值 `threshold`: 0.2/0.3/0.4/0.5
4. 人脸质量过滤：`blur_threshold`, `min_face_size`

输出：
1. 精度-召回变化
2. 查询耗时变化
3. 最优参数区间

---

## 3.4 实验 D：工程效率实验（老师常问）

目标：说明系统可落地，不只是“能跑”。

记录：
1. 建索引耗时（按数据规模）
2. 单 query 平均耗时（P50/P95）
3. 索引大小（MB）
4. 特征维度与速度关系（512 vs 2048，可选）

展示形式：
1. 数据规模 N 与耗时的折线图
2. N 与内存占用/索引文件大小关系

---

## 3.5 实验 E：跨域泛化实验（答辩杀手锏）

目标：证明模型不是只在同源数据“记住了背景”。

设置：
1. 在公开数据上训练/选参数（如 `Market-1501` for person）
2. 在你自建视频数据（`yoasobi_TFT`, `egodeath`）上直接测试
3. 比较“训练域内 vs 训练域外”的性能差

解释：
1. 域外下降是正常现象
2. 你可以用轻量微调或阈值校准缓解下降

## 4. 指标口径（建议统一）

1. `Rank-1`：Top1 是否命中同身份
2. `Recall@K`：TopK 内是否至少命中 1 个同身份
3. `AP`：单 query 的平均精度
4. `mAP`：所有 query 的 AP 平均

建议同时报告：
1. macro（按身份平均）
2. micro（按样本平均）

原因：你的数据类别不平衡，macro 更公平。

## 5. 你最关心的问题：全是预训练，如何讲“贡献”？

## 5.1 正确叙事方式

不要把贡献说成“我发明了 ArcFace/ResNet”，而是说：

1. 我们提出并实现了 **双分支人物检索框架**（face/person）
2. 我们设计了 **质量感知与参数消融评测协议**
3. 我们在自建视频数据上做了 **跨域验证与误差分析**
4. 我们完成了 **可复现实验工程**（索引构建、检索、结果落盘、指标统计）

这在课程答辩里是成立且专业的。

## 5.2 你可以做的“算法贡献”最小闭环

1. 做一个融合策略（线性加权 + 动态权重）
   - 例：人脸质量高时增大 face 权重；低质量时增大 person 权重
2. 做一个重排策略（TopK 二阶段）
   - 第一阶段全量召回，第二阶段融合重排
3. 用消融证明每个模块的增益
   - baseline -> +融合 -> +动态权重 -> +重排

只要表格能证明提升，就是有效贡献。

## 6. 是否要自己训练模型？建议与路线

## 6.1 推荐路线（优先）

训练 `person` 分支的 ResNet（而不是替掉 ArcFace）：

1. 用 `Market-1501` 训练 `ResNet50` ReID 特征（CE + Triplet）
2. 输出 checkpoint：`models/weights/person_resnet50_finetuned.pt`
3. 在当前项目里通过 `--resnet-weight-path` 接入
4. 在 `egodeath/yoasobi_TFT` 做泛化测试

优点：
1. 与你项目当前结构强一致（代码已有接口）
2. 可清晰声明“这是我们自己训练的核心模型”
3. 工作量可控，答辩可解释性强

## 6.2 不太建议路线（除非算力充足）

1. 从零训练一个“替代 ArcFace”人脸模型
2. 用很小的人脸数据直接重训大 backbone

风险：
1. 训练成本高
2. 结果未必超过 ArcFace
3. 很容易在答辩时被问到“大模型对比与公平性”

## 6.3 折中路线（也很好讲）

1. ArcFace 保持预训练作为强基线
2. 你只微调一个小投影头/分类头（或仅做阈值校准）
3. 强调“我们贡献是检索方法和系统，不是重造通用人脸 backbone”

## 7. 可直接执行的最小实验清单（建议 2 周版本）

1. 第 1-2 天：重划分 `yoasobi_TFT/egodeath` 的 gallery/query，保证每身份 query 数足够
2. 第 3-4 天：跑 `face-only` 与 `person-only` 基线，得到 Rank-1/Recall@K/mAP
3. 第 5-6 天：做阈值与采样率消融，补效率指标
4. 第 7-10 天：训练 `person` 的 ResNet checkpoint 并接入对比
5. 第 11-12 天：做鲁棒性退化实验
6. 第 13-14 天：整理答辩图表与错误案例
