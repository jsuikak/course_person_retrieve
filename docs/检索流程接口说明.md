# 应用级检索流程接口说明（face/person）

## 1. 目标

提供应用级流程入口，串联：

1. 建索引（索引存在则跳过）
2. 执行检索（复用 `src/retrieval.py`）

并支持 `feature_mode` 切换：

1. `face`：MTCNN + ArcFace
2. `person`：YOLO + ResNet

---

## 2. 代码位置

- `src/app_retrieval.py`
- 核心函数：`run_app_retrieval_flow(...)`
- CLI 入口：`uv run python -m src.app_retrieval ...`

---

## 3. 函数签名（关键参数）

```python
run_app_retrieval_flow(
    query_path: str,
    gallery_path: str,
    index_name: str | None = None,
    topk: int = 5,
    arcface_weight_path: str = "./models/weights/arcface.pt",
    device: str = "cpu",
    indexes_root: str = "indexes",
    retrieval_output_root: str = "outputs/retrieval",
    feature_mode: Literal["face", "person"] = "face",
    yolo_weights: str = "yolo11n.pt",
    yolo_conf: float = 0.25,
    yolo_iou: float = 0.7,
    yolo_max_det: int = 100,
) -> dict[str, Any]
```

---

## 4. 流程规则

1. `index_name` 未传时，默认取 `gallery_path` 的文件/目录名。
2. 自动识别 `gallery_path` 类型（image/video）并选择索引目录：
   - 图像库 -> `indexes/image_index/`
   - 视频库 -> `indexes/video_index/`
3. 索引命名统一为：
   - `<index_name>_face_{features,meta,info}`
   - `<index_name>_person_{features,meta,info}`
4. 若三文件都存在，则跳过建索引。
5. 不存在则调用 `build_feature_index(...)` 构建索引（`build_face_feature_index(...)` 仍可兼容调用）。
6. 最后调用 `search_query_in_index(...)` 执行检索并落盘结果。

---

## 5. CLI 示例

### 5.1 face 模式

```bash
uv run python -m src.app_retrieval \
  --query data_runtime/query/ikura.jpg \
  --gallery data_runtime/gallery/images/yoasobi_TFT \
  --feature-mode face \
  --topk 5 \
  --weights ./models/weights/arcface.pt \
  --device cpu
```

### 5.2 person 模式

```bash
uv run python -m src.app_retrieval \
  --query data_runtime/query/ikura.jpg \
  --gallery data_runtime/gallery/images/yoasobi_TFT \
  --feature-mode person \
  --yolo-weights yolo11n.pt \
  --yolo-conf 0.25 \
  --yolo-iou 0.7 \
  --yolo-max-det 100 \
  --topk 5 \
  --weights ./models/weights/arcface.pt \
  --device cpu
```

输出 JSON 摘要包含：

1. `build.status`：`built` 或 `skipped`
2. `build.feature_mode`：`face` 或 `person`
3. `build.index_paths`：索引文件路径
4. `retrieval.result_json`：检索结果文件路径
5. `retrieval.output_dir`：检索产物目录（含 `crops/`、`annotated/`）
