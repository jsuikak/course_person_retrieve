# 应用级检索流程接口说明

## 1. 目标

提供应用级流程入口，串联：

1. 建索引（索引存在则跳过）
2. 执行检索（复用 `src/retrieval.py`）

---

## 2. 代码位置

- `src/app_retrieval.py`
- 核心函数：`run_app_retrieval_flow(...)`
- CLI 入口：`python3 -m src.app_retrieval ...`

---

## 3. 函数签名

```python
run_app_retrieval_flow(
    query_path: str,
    gallery_path: str,
    index_name: str | None = None,
    topk: int = 5,
    arcface_weight_path: str = "./models/weights/arcface.pt",
    device: str = "cpu",
    indexes_root: str = "indexes",
    retrieval_output_root: str = "outputs/retrieval",
) -> dict[str, Any]
```

---

## 4. 流程规则

1. `index_name` 未传时，默认取 `gallery_path` 的文件/目录名。
2. 自动识别 `gallery_path` 类型（image/video）并选择索引目录：
   - 图像库 -> `indexes/image_index/`
   - 视频库 -> `indexes/video_index/`
3. 若以下三文件都存在，则跳过建索引：
   - `<index_name>_features.npy`
   - `<index_name>_meta.csv`
   - `<index_name>_info.json`
4. 不存在则调用 `build_face_feature_index(...)` 构建索引。
5. 最后调用 `search_query_in_index(...)` 执行检索并落盘结果（仅在对应索引子目录检索，避免同名冲突）。

---

## 5. CLI 示例

```bash
python3 -m src.app_retrieval \
  --query data_runtime/query/ikura.jpg \
  --gallery data_runtime/gallery/images/yoasobi_TFT \
  --topk 5 \
  --weights ./models/weights/arcface.pt \
  --device cpu
```

输出为 JSON 摘要，包含：

1. `build.status`：`built` 或 `skipped`
2. `build.index_paths`：索引文件路径
3. `retrieval.result_json`：检索结果文件路径
4. `retrieval.output_dir`：检索产物目录（含 `crops/`、`annotated/`）
