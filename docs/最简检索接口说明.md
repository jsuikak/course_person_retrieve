# 最简检索接口说明（face/person）

## 1. 目标

提供一个最简函数接口，只接收：

1. `query` 图片路径
2. `索引名称`
3. `feature_mode`（`face` 或 `person`）

函数会自动读取索引、执行匹配、并将 TopK 可视化结果输出到检索结果目录。

---

## 2. 代码位置

- `src/retrieval.py`
- 入口函数：`search_query_in_index(...)`

---

## 3. 函数签名（关键参数）

```python
search_query_in_index(
    query_path: str,
    index_name: str,
    topk: int = 5,
    arcface_weight_path: str = "./models/weights/arcface.pt",
    device: str = "cpu",
    indexes_root: str = "indexes",
    retrieval_output_root: str = "outputs/retrieval",
    feature_mode: Literal["face", "person"] = "face",
    yolo_weights: str = "yolo11n.pt",
    yolo_conf: float = 0.25,
    yolo_iou: float = 0.7,
    yolo_max_det: int = 100,
) -> dict[str, Any]
```

---

## 4. 输入规则

1. `query_path`：查询图片路径。
2. `index_name`：索引名（通常等于库名）。
3. 仅支持新命名索引：
   - `<index_name>_face_features.npy` / `<index_name>_face_meta.csv`
   - `<index_name>_person_features.npy` / `<index_name>_person_meta.csv`

---

## 5. 模式差异

### 5.1 face 模式

1. query 图做人脸检测，可多脸。
2. 对每条库特征取“与 query 所有人脸的最大分”作为最终得分。

### 5.2 person 模式（person-only）

1. query 图默认按整图提 1 条 person 特征（不做检测）。
2. gallery 建索引阶段做人检测并多框入库。

---

## 6. 输出规则

输出目录固定为：

`outputs/retrieval/<query文件名>-<索引名>-<feature_mode>/`

目录内容：

1. `results.json`：TopK 结果（分数、来源、bbox、落盘路径等）
2. `crops/`：每个命中结果的裁剪图
3. `annotated/`：每个命中结果的原图叠框图（检测框上标注 `query` 名称）

说明：

1. 每次执行前会清空同名输出目录，防止历史结果残留导致新旧结果混杂。
