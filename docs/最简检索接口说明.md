# 最简检索接口说明

## 1. 目标

提供一个最简函数接口，不区分 `face/person` 输入模式，只接收：

1. `query` 图片路径
2. `索引名称`（按库名）

函数会自动读取索引、执行匹配、并将 TopK 可视化结果输出到检索结果目录。

---

## 2. 代码位置

- `src/retrieval.py`
- 入口函数：`search_query_in_index(...)`

---

## 3. 函数签名

```python
search_query_in_index(
    query_path: str,
    index_name: str,
    topk: int = 5,
    arcface_weight_path: str = "./models/weights/arcface.pt",
    device: str = "cpu",
    indexes_root: str = "indexes",
    retrieval_output_root: str = "outputs/retrieval",
) -> dict[str, Any]
```

---

## 4. 输入规则

1. `query_path`：查询图片路径。
2. `index_name`：索引名（通常等于库名）。
3. 程序在 `indexes_root` 下查找：
   - `<index_name>_features.npy`
   - `<index_name>_meta.csv`

例如索引名为 `yoasobi`，则程序查找 `**/yoasobi_features.npy` 和同目录下的 `yoasobi_meta.csv`。

---

## 5. 输出规则

输出目录固定为：

`outputs/retrieval/<query文件名>-<库名>/`

目录内容：

1. `results.json`：TopK 结果（分数、来源、bbox、落盘路径等）
2. `crops/`：每个命中结果的裁剪图
3. `annotated/`：每个命中结果的原图叠框图（检测框上标注 `query` 名称）

说明：

1. 每次执行前会清空同名输出目录，防止历史结果残留导致新旧结果混杂。

---

## 6. 多人脸处理

1. query 图中可检测出多个人脸。
2. 对库中每一条特征，计算其与“所有 query 人脸特征”的相似度。
3. 取该库特征的最大相似度作为最终分数。
4. 按该分数全局排序并返回 TopK。

即：输出是“TopK 个相似度最高的库特征”，不是“每张图仅一条”。

---

## 7. 关于 bbox 的问题

当前 `src` 索引实现包含 bbox 信息，不缺失：

1. 特征文件：`<prefix>_features.npy`
2. 元数据文件：`<prefix>_meta.csv`
3. `meta.csv` 字段包含：`x, y, w, h`（以及 `source_name, frame_index, face_index`）

因此可以完成：

1. 命中框裁剪（`crops/`）
2. 原图叠框并标注 query 名称（`annotated/`）

---

## 8. 最小调用示例

```python
from src.retrieval import search_query_in_index

out = search_query_in_index(
    query_path="data_runtime/query/ikura_0001.jpg",
    index_name="yoasobi",
    topk=5,
    arcface_weight_path="./models/weights/arcface.pt",
    device="cpu",
)
print(out)
```
