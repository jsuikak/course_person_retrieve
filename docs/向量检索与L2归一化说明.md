# 向量检索与 L2 归一化说明

## 1. 当前检索方案概览

本项目当前的向量检索方案可以概括为：

1. 先把图库或视频库中的目标区域提取成特征向量。
2. 将所有特征向量堆成一个二维矩阵，并和元数据一起落盘。
3. 检索时读取该矩阵，对 query 特征和 gallery 特征做 L2 normalize。
4. 用矩阵点积计算余弦相似度。
5. 按相似度从高到低排序，取 Top-K，并输出命中裁剪图、标注图和 `results.json`。

需要注意：这里的“索引”不是 FAISS、Milvus 这类向量数据库或近似最近邻结构，而是一个持久化的特征矩阵加元数据。检索阶段会用 NumPy 对整个 gallery 矩阵做一次全量相似度计算。

核心代码位置：

1. 应用级入口：`src/app_retrieval.py`
2. 建索引：`src/face_index_builder.py`
3. 检索与结果落盘：`src/retrieval.py`
4. 特征提取：`src/tools/feature_extractor.py`
5. 通用特征匹配：`src/tools/feature_matcher.py`

## 2. 两条特征链路

当前支持 `feature_mode=face` 和 `feature_mode=person` 两种模式。

### 2.1 face 模式

图库或视频库：

1. 读取图片或视频帧。
2. 用 MTCNN 检测人脸框。
3. 按检测框裁剪人脸区域。
4. 将裁剪图 resize 到 ArcFace 输入尺寸。
5. 用 ArcFace 提取 512 维人脸 embedding。
6. 对 embedding 做 L2 normalize 后保存。

query：

1. 当前实现中 query 阶段不再检测人脸，而是把整张 query 图作为输入。
2. ArcFace 提取 512 维向量。
3. 如果开启 flip test，会分别提取原图和水平翻转图特征，取平均后再做一次 L2 normalize。

因此，`face` 模式下 query 图片最好已经是目标人脸较清楚、占比较高的图。若 query 图包含多个人脸或大量背景，当前实现不会自动选出其中某一张脸。

### 2.2 person 模式

图库或视频库：

1. 读取图片或视频帧。
2. 用 YOLO 检测人体框。
3. 按人体框裁剪 person 区域。
4. 用 ResNet 提取人体外观特征。
5. 对 embedding 做 L2 normalize 后保存。

query：

1. 当前实现中 query 阶段不做人检测，而是把整张 query 图作为输入。
2. 用 ResNet 提取一个 person 特征。
3. 对该特征做 L2 normalize 后参与检索。

默认 `resnet_backbone=resnet50` 时，person 特征维度是 2048；如果切换到 `resnet18` 或 `resnet34`，维度是 512。

## 3. 索引文件是什么

建索引后，每个索引会落盘三类文件：

1. `<index_name>_<feature_mode>_features.npy`
2. `<index_name>_<feature_mode>_meta.csv`
3. `<index_name>_<feature_mode>_info.json`

其中最关键的是 `features.npy`，它是一个形状为 $(N, D)$ 的 `float32` 矩阵：

$$
G =
\begin{bmatrix}
g_1 \\
g_2 \\
\vdots \\
g_N
\end{bmatrix}
$$

含义：

1. $N$ 是 gallery 中成功提取到的目标数量。对 face 是人脸数量，对 person 是人体框数量。
2. $D$ 是特征维度。face 通常是 512，person 默认 ResNet50 是 2048。
3. 第 $i$ 行 $g_i$ 对应 `meta.csv` 第 $i$ 行元数据。
4. `meta.csv` 记录来源文件、视频帧号、检测框坐标等信息，用于把向量检索结果映射回原图或视频帧。

## 4. 检索时如何打分

检索时会得到 query 特征矩阵：

$$
Q =
\begin{bmatrix}
q_1 \\
\vdots \\
q_M
\end{bmatrix}
$$

当前 query 实际通常只有一条特征，即 $M=1$。

然后对 gallery 和 query 都做 L2 normalize：

$$
\hat{g}_i = \frac{g_i}{\lVert g_i \rVert_2}
$$

$$
\hat{q}_j = \frac{q_j}{\lVert q_j \rVert_2}
$$

再计算点积：

$$
\operatorname{score}(i, j) = \hat{g}_i \cdot \hat{q}_j
$$

因为两边都已经是单位向量，所以这个点积就是余弦相似度：

$$
\operatorname{score}(i, j) = \cos(\theta)
$$

其中 $\theta$ 是两个向量的夹角。分数越大，说明方向越接近，系统认为两者越相似。

当前排序逻辑：

1. `person` 模式：每个 gallery person 特征与 query person 特征计算相似度。
2. `face` 模式：代码支持对多个 query face 特征取最大相似度，但当前 query 阶段实际只产出一个特征，所以等价于单 query 特征匹配。
3. 取分数最高的前 `topk` 个 gallery 行号。
4. 根据 `meta.csv` 回读来源图片或视频帧，裁剪命中框，并保存标注结果。

## 5. L2 normalize 是什么

对一个向量：

$$
x = [x_1, x_2, \ldots, x_D]
$$

它的 L2 范数是：

$$
\lVert x \rVert_2 = \sqrt{x_1^2 + x_2^2 + \cdots + x_D^2}
$$

L2 normalize 就是把向量除以自己的 L2 范数：

$$
\hat{x} = \frac{x}{\lVert x \rVert_2}
$$

归一化后：

$$
\lVert \hat{x} \rVert_2 = 1
$$

也就是说，处理后的向量都落在一个 $D$ 维单位超球面上。它们仍然是实数向量，分量可以为正、为负或为零；它们不是概率分布，分量之和不需要等于 1。L2 normalize 约束的是“平方和为 1”：

$$
\sum_i \hat{x}_i^2 = 1
$$

## 6. 归一化后向量的数学特征

L2 normalize 后的向量有几个关键特征。

第一，长度固定为 1。

所有非零向量都会被投影到单位球面上。归一化后，向量的模长不再参与相似度比较。

第二，方向保持不变。

对于正数 $c > 0$：

$$
\operatorname{normalize}(c x) = \operatorname{normalize}(x)
$$

这表示如果两个向量只是整体尺度不同，它们会被视为同一个方向。

第三，点积等于余弦相似度。

未归一化时：

$$
x \cdot y = \lVert x \rVert_2 \lVert y \rVert_2 \cos(\theta)
$$

点积同时受长度和夹角影响。归一化后：

$$
\hat{x} \cdot \hat{y} = \cos(\theta)
$$

此时分数只反映方向接近程度，不再被向量长度放大或压低。

第四，在单位球面上，余弦相似度和欧氏距离排序等价。

对两个单位向量 $u$ 和 $v$：

$$
\lVert u - v \rVert_2^2 = 2 - 2(u \cdot v)
$$

所以点积越大，欧氏距离越小。也就是说，在已经 L2 normalize 的前提下，用点积找最大相似度，和用 L2 距离找最近邻在排序上是一致的。

## 7. 为什么当前方案需要 L2 normalize

不是所有向量都天然必须做 L2 normalize。这里要做，是因为当前系统把 embedding 的“方向”作为身份或外观相似性的主要依据，并且后续用余弦相似度排序。

### 7.1 防止向量长度干扰相似度

如果不做归一化，点积是：

$$
x \cdot y = \lVert x \rVert_2 \lVert y \rVert_2 \cos(\theta)
$$

这会导致一个问题：即使两个向量夹角一样，范数更大的向量也会得到更高点积分数。

例子：

$$
\text{候选 A: } \lVert g_A \rVert_2 = 10,\ \cos(\theta)=0.70
\Rightarrow g_A \cdot q \approx 7.0
$$

$$
\text{候选 B: } \lVert g_B \rVert_2 = 2,\ \cos(\theta)=0.95
\Rightarrow g_B \cdot q \approx 1.9
$$

如果直接按点积排序，A 会排在 B 前面，但 B 的方向其实更接近 query。对人物检索来说，这通常不是我们想要的结果。

L2 normalize 后：

$$
\operatorname{score}_A = 0.70
$$

$$
\operatorname{score}_B = 0.95
$$

排序会回到“方向更接近者更相似”。

### 7.2 ArcFace 本身就是角度判别思路

`face` 模式使用 ArcFace。ArcFace 的核心思想就是让不同身份在特征空间里按角度分开，同身份特征方向更接近，不同身份特征方向更远。因此做人脸检索时，常见做法就是比较归一化 embedding 的余弦相似度或角距离。

当前代码也符合这个思路：

1. ArcFace 输出后先做 L2 normalize。
2. flip test 平均后再做 L2 normalize。
3. 检索阶段对 gallery 和 query 又做一次 L2 normalize。
4. 最终用点积作为 `score`。

其中第 3 步是重复归一化，但它基本是幂等的：一个已经单位化的非零向量再次单位化，结果几乎不变。它的好处是让检索函数对旧索引、外部导入向量或数值误差更稳健。

### 7.3 person 特征也需要统一到同一尺度

`person` 模式使用 ResNet 提取人体外观特征。ResNet 输出的原始特征长度可能受到图像亮度、裁剪尺度、背景、模型激活强弱等因素影响，这些长度差异不一定代表“是不是同一个人”。

L2 normalize 后，检索只比较方向，能减少特征模长对排序的影响。需要同时注意：当前 person 特征是否足够适合人物重识别，取决于 ResNet 权重本身。如果只是普通分类预训练或随机权重，L2 normalize 可以让打分定义更稳定，但不能替代一个真正训练好的 ReID 特征模型。

## 8. 当前实现中的细节和边界

1. 特征提取阶段已经做归一化：`FeatureExtractor._face_forward()` 和 `_extract_person_from_bgr()` 都会返回 L2-normalized 特征。
2. 检索阶段再次归一化：`search_query_in_index()` 会对 `gallery_features` 和 `query_features` 调用 `_l2_normalize_rows()`。
3. 通用匹配器也再次归一化：`src/tools/feature_matcher.py` 中的 `match()` 会对单个 query 和 gallery 矩阵归一化后再点积。
4. 代码里使用 `eps=1e-12` 处理接近零范数的向量，避免除零。真实神经网络 embedding 通常不会是零向量，这更多是防御性处理。
5. `score` 是特征相似度，不是 MTCNN 或 YOLO 的检测置信度。
6. 理论上余弦相似度范围是 $[-1, 1]$。实际检索中，高相似通常接近 1，低相似可能接近 0 或为负，具体阈值需要结合数据集实验标定。

## 9. 一句话回答疑问

L2 normalize 的本质是把每个 embedding 变成长度为 1 的单位向量，让后续点积只比较向量方向，也就是余弦相似度。当前 face/person 检索都把“方向是否接近”当作相似性的主要依据，所以需要先消除向量长度带来的干扰。归一化后的向量数学上位于单位超球面，满足 $\sum_i x_i^2 = 1$，但它们不是概率向量，也不是每个分量都非负。
